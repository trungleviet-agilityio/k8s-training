# Related Exercises

## Phase 2 Exercises

The following exercises are available in the `exercises/` directory to practice the concepts covered in this phase.

| Topic | Directory | Description |
|-------|-----------|-------------|
| **Pods** | `exercises/pods/` | Basic pod creation, static pods, and simple containers. |
| **Deployments** | `exercises/deployments/` | Creating deployments, scaling, rolling updates, and rollbacks. |
| **ConfigMaps & Secrets** | `exercises/configmaps-secrets/` | Managing configuration and sensitive data using ConfigMaps and Secrets. |
| **Probes** | `exercises/probes/` | Implementing Liveness, Readiness, and Startup probes for application health. |
| **Multi-container Pods** | `exercises/multi-container/` | Implementing sidecar patterns for logging or monitoring. |
| **Jobs & CronJobs** | `exercises/jobs-cronjobs/` | Running batch jobs and scheduled tasks. |
| **API Deployment** | `exercises/api-deployment/` | A complete example combining deployments, config, secrets, and probes. |
| **Debugging** | `exercises/debugging/` | Scenarios to practice troubleshooting broken pods and deployments. |

## Recommended Practice Path

1. **Start with Pods:** Create simple pods to understand the basic unit of Kubernetes.
2. **Move to Deployments:** Learn how to manage application lifecycles and scaling.
3. **Add Configuration:** Externalize configuration using ConfigMaps and Secrets.
4. **Ensure Health:** Add probes to make your application robust.
5. **Advanced Patterns:** Try multi-container pods and batch jobs.
6. **Put it Together:** Implement the full API Deployment example.
7. **Challenge:** Try the debugging exercises to test your troubleshooting skills.
